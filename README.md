# 🩸 Кровавый Меридиан — игровой форум

Полноценный игровой форум: Node.js (Express) + SQLite на бэкенде, чистый HTML/CSS/JavaScript
на фронтенде (без фреймворков и сборщиков — просто открыли и работает). Тёмно-синий игровой
дизайн с неоновыми голубыми акцентами, стеклянными карточками и анимациями.

## 🚀 Быстрый запуск

```bash
cd server
npm install
npm start
```

Откройте **http://localhost:5000** — сервер раздаёт и API, и клиентскую часть с одного адреса.

> ⚠️ Требуется **Node.js версии 22.5 или выше** — проект использует встроенный модуль
> `node:sqlite`, поэтому СУБД **не требует нативной компиляции** и устанавливается мгновенно
> в любой песочнице/контейнере без доступа к интернету за пределами npm-реестра.
> Проверить версию: `node -v`.

### Учётная запись администратора по умолчанию

При первом запуске база данных создаётся автоматически и заполняется тестовыми категориями
и администратором:

```
email:    admin@meridian.local
ник:      Admin
пароль:   Admin12345!
```

**Обязательно смените этот пароль после первого входа в реальном проекте** (Профиль → Безопасность).

### Роль «Команда Проекта» (владелец проекта)

Высшая роль форума — выше администратора. Участник Команды Проекта может управлять кем угодно,
включая снятие/блокировку администраторов. Эту роль **нельзя выдать через интерфейс** — только
редактированием `server/.env`:

```
OWNER_EMAILS=owner@meridian.local,founder@meridian.local
```

Любой пользователь, который входит в аккаунт (или регистрируется) с email из этого списка,
автоматически получает роль «Команда Проекта» при следующем входе.

### Настройка почты (необязательно)

По умолчанию письма (подтверждение email, сброс пароля) не отправляются на реальную почту,
а красиво печатаются в консоль сервера со ссылкой внутри — этого достаточно для разработки
и тестирования. Чтобы подключить настоящую отправку писем, скопируйте `server/.env.example`
в `server/.env` и заполните поля `SMTP_HOST`, `SMTP_USER`, `SMTP_PASS` (подойдёт, например,
Gmail с паролем приложения или бесплатный Mailtrap.io для тестов).

## 📂 Структура проекта

```
/client                  — статичный фронтенд (HTML/CSS/JS, без сборки)
  index.html              — главная страница (баннер, категории, новости)
  forum.html               category.html      — список категорий / список тем в категории
  topic.html                new-topic.html     — просмотр темы (ответы, цитаты, лайки) / создание темы
  news.html                 news-item.html     — лента новостей / отдельная новость
  users.html                profile.html       — каталог пользователей / профиль (свой и чужой)
  messages.html              support.html      — личные сообщения / поддержка (FAQ + форма)
  login.html  register.html  verify.html        — вход, регистрация, подтверждение email
  forgot-password.html  reset-password.html     — восстановление пароля
  admin.html                search.html         — админ-панель / результаты поиска
  style.css                — вся стилистика (переменные темы, компоненты, адаптивность)
  app.js                    — общий модуль: API-клиент, шапка/футер, тосты, уведомления, авторизация
  /js/*.js                 — скрипт под каждую страницу (login.js, topic.js, admin.js, …)

/server
  server.js                — точка входа, мидлвары, статика, маршруты, обработка ошибок
  /database/db.js          — схема SQLite (node:sqlite), сидирование начальных данных
  /routes/*.js              — маршруты Express по доменам (auth, users, forum, admin, news, messages, notifications, support)
  /controllers/*.js         — бизнес-логика обработчиков
  /middleware
     auth.js                — JWT-проверка (requireAuth / optionalAuth / requireRole)
     upload.js               — загрузка аватаров через multer
     validate.js             — обработка результатов express-validator
  /utils
     mailer.js              — отправка email (nodemailer либо лог в консоль)
     helpers.js              — publicUser(), пагинация, онлайн-статус
  /uploads/avatars          — загруженные аватары (+ дефолтная SVG-заглушка)
  package.json  .env.example
```

## ✅ Реализованный функционал

**Регистрация и авторизация**
- Регистрация по email с придумыванием ника/email/пароля, проверка уникальности обоих полей
- Подтверждение email по ссылке из письма (или из консоли в режиме разработки)
- Восстановление пароля по email (ссылка с истечением через 1 час)
- Вход по email **или** нику, чекбокс «Запомнить меня» (срок жизни JWT 1 день / 30 дней)
- Выход из аккаунта

**Профиль**
- Аватар (загрузка файла), никнейм, статус, дата регистрации, число сообщений/тем, репутация
- Редактирование аватара, ника, пароля, описания «о себе»
- Просмотр чужих профилей с переходом к личным сообщениям

**Форум**
- 6 категорий: «Новости проекта», «Помощь игрокам» и «Гайды» закреплены первыми и доступны
  для создания тем/ответов только персоналу (модератор/админ/Команда Проекта); «Общение»,
  «Баги и ошибки», «Предложения» — открыты для всех
- Список тем с количеством сообщений, просмотров и последним ответом
- Создание / редактирование / удаление своей темы, ответы, цитирование чужих сообщений, лайки
  (лайк увеличивает репутацию автора сообщения)
- Редактировать текст темы/сообщения может **только автор** — даже администратор не может
  подменить чужие слова; для модерации недопустимого контента персонал может только удалить
  тему/сообщение целиком, не редактируя их
- Полнотекстовый поиск по темам, сообщениям и пользователям

**Роли и иерархия**
- `user` (Пользователь) → `moderator` (Модератор) → `admin` (Администратор) → `owner` (Команда Проекта)
- Модератор: модерация контента (закрепление/закрытие/удаление тем), но **не может банить пользователей**
- Администратор: может банить и менять роли пользователей/модераторов, но не трогает других
  администраторов и Команду Проекта
- Команда Проекта: полный доступ, включая блокировку/смену ролей администраторов. Выдаётся
  только через `.env` (см. выше) — недоступна через интерфейс ни одной роли, включая админа

**Админ-панель**
- Блокировка/разблокировка пользователей с указанием причины (только admin/owner)
- Выдача ролей с учётом иерархии (только admin/owner)
- Удаление, закрепление и закрытие тем (admin/moderator/owner)
- Управление новостями
- Сводная статистика (пользователи, темы, сообщения, онлайн)

**Дополнительно**
- Список пользователей онлайн (активность за последние 5 минут); выход из аккаунта мгновенно
  сбрасывает статус «онлайн» на сервере (раньше статус держался до 5 минут после выхода)
- Уведомления (ответы, цитаты, лайки, личные сообщения, системные — например, смена роли)
- Личные сообщения между пользователями с непрочитанными счётчиками; форма «Поддержка» для
  авторизованных пользователей создаёт настоящий личный диалог с администратором (можно получить
  ответ и продолжить переписку в разделе «Сообщения»), а не просто разовое уведомление
- Полная адаптивность под телефон (бургер-меню, одноколоночная сетка)
- Защита от SQL-инъекций (параметризованные запросы везде), хеширование паролей bcrypt,
  JWT-авторизация, серверная валидация (express-validator), базовый rate-limiting
- Собственные SVG/PNG-иконки проекта (логотип, иконки разделов, бейджи «закреплено»/«закрыто»)
  вместо эмодзи — сгенерированы программно и лежат в `client/icons/`

## 🛢 О выборе базы данных

Изначально планировался пакет `better-sqlite3`, но он требует нативной компиляции под
конкретную платформу. Чтобы проект гарантированно запускался «из коробки» в любой среде
(включая офлайн/изолированные песочницы), бэкенд использует **встроенный в Node.js модуль
`node:sqlite`** (стабилен с экспериментальным флагом начиная с Node 22.5). Слой доступа к
данным целиком изолирован в `server/database/db.js` — чтобы перейти на **MySQL**, замените
этот файл на обёртку над `mysql2` с тем же набором функций (`get`, `all`, `run`) и поправьте
несколько SQL-отличий (`AUTOINCREMENT` → `AUTO_INCREMENT`, синтаксис `datetime('now')` и т.д.);
остальной код (контроллеры, маршруты) трогать не придётся.

## 🎨 О дизайне

Тёмно-синий градиентный фон, неоновые голубые кнопки, полупрозрачные «стеклянные» карточки
(`backdrop-filter: blur`), закругления 15–20px, мягкие тени и плавные анимации — все переменные
темы вынесены в `:root` в начале `style.css`, так что цветовую схему легко перенастроить под
свой проект.
